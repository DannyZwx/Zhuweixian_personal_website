# 朱炜贤 HR 个人网站 V55 使用说明（含 V1-V55 更新日志）


====================
一、压缩包目录结构
====================

压缩包内主要包含两个文件夹：

1. `website/`
   这是正式网站文件夹。上传域名服务器时，只上传这个文件夹里的内容。

2. `editor_tool/`
   这是本地可视化编辑器。它只用于在电脑本地修改网站内容，不要上传到域名服务器。

常见路径：

- 网站首页：`website/index.html`
- 网站主页：`website/home.html`
- 网站数据文件：`website/assets/js/site-data.js`
- 网站主脚本：`website/assets/js/main.js`
- 隐藏页脚本：`website/assets/js/avatar-secret.js`
- 编辑器入口：`editor_tool/editor.html`
- 编辑器脚本：`editor_tool/editor.js`


====================
二、如何预览网站
====================

方式一：直接打开

双击打开：

`website/index.html`

适合快速查看前页、主页、导航、作品集、经历、简历、网站制作花絮等页面效果。

方式二：通过编辑器预览

打开：

`editor_tool/editor.html`

在编辑器内修改内容后，可以使用编辑器里的保存、预览、导出等功能进行本地检查。

注意：部分浏览器会限制本地文件的读写能力。如果编辑器提示绑定网站文件夹，请选择压缩包里的 `website/` 文件夹。

====================
四、编辑后如何保存与发布
====================

本地保存流程：

1. 打开 `editor_tool/editor.html`。
2. 绑定 `website/` 文件夹。
3. 在左侧选择要修改的模块。
4. 修改文字、图片路径、音频路径或 iframe 路径。
5. 点击保存。
6. 打开或刷新 `website/index.html` / `website/home.html` 检查效果。

正式上传域名流程：

1. 在编辑器中完成所有修改。
2. 确认 `website/assets/js/site-data.js` 已更新。
3. 只上传 `website/` 文件夹中的内容到服务器。
4. 不要上传 `editor_tool/` 文件夹。
5. 如果网站没有变化，尝试清理浏览器缓存，或按 `Ctrl + F5` 强制刷新。

常见服务器目录示例：

如果你的域名根目录是：

`/www/wwwroot/你的域名/`

那么应上传：

- `website/index.html` 到 `/www/wwwroot/你的域名/index.html`
- `website/home.html` 到 `/www/wwwroot/你的域名/home.html`
- `website/assets/` 到 `/www/wwwroot/你的域名/assets/`
- `website/pages/` 到 `/www/wwwroot/你的域名/pages/`

不要上传成：

`/www/wwwroot/你的域名/website/index.html`

否则访问域名时可能找不到首页。


====================
五、素材放置规范
====================

所有图片建议放在：

`website/assets/images/`

所有音频建议放在：

`website/assets/audio/`

Godot Web 导出文件建议放在：

`website/assets/godot/你的游戏名/`

推荐素材规格：

- 头像 / 角色形象：建议 4:5 或 3:4，推荐 1200×1500px / 1200×1600px。
- 蛋仔透明 PNG 形象：建议 1:1 或 4:5，推荐 1200×1200px 透明 PNG。
- 软件图标：建议 1:1，推荐 512×512px，PNG/SVG。
- 游戏图标：建议 1:1，推荐 512×512px。
- 视频 / 小游戏封面：建议 16:9，推荐 1920×1080px。
- 编曲作品封面：建议 1:1，推荐 1200×1200px。
- 平面作品图片：支持 3:4、4:5、1:1、16:9，建议宽度不少于 1200px。
- 校园经历 / 项目经历详情图片：建议 4:3 或 16:9，推荐 1600×1200px / 1920×1080px。
- 获奖证书图片：建议 4:3 或 A4 竖版比例，推荐长边 1600px 以上。
- 网站制作花絮流程节点图片：建议 16:9 或 4:3，推荐 1600×900px / 1600×1200px。
- 隐藏页鼠标拖尾图片：建议透明 PNG 或主题协调图片，推荐 1:1 或 4:5。

路径填写示例：

`assets/images/avatar.png`

不要写成电脑绝对路径，例如：

`C:\Users\你的名字\Desktop\avatar.png`

网页上线后无法读取这种本机路径。


====================
六、Godot 小游戏嵌入方法
====================

Godot 导出 Web 后，通常会得到：

- `index.html`
- `.wasm`
- `.pck`
- `.js`

请把整个导出文件夹放入：

`website/assets/godot/你的游戏名/`

例如：

`website/assets/godot/0618/index.html`

然后在编辑器的“小游戏作品”里填写：

`assets/godot/0618/index.html`

注意：Godot Web 游戏上线后可能需要服务器正确支持 `.wasm` 文件。如果本地能打开，但上线后打不开，要检查服务器 MIME 类型和跨域限制。


====================
七、隐藏页使用说明
====================

隐藏页入口：

在主页长按头像 / 角色形象卡片，会出现圆形进度提示。进度满后进入隐藏页。

V36 当前逻辑：

- 长按进入速度已加快，约 620ms。
- 隐藏页背景使用原网站背景逻辑，并加入等距网格效果。
- 鼠标移动时会触发图片拖尾彩蛋。
- 拖尾图片会在页面上停留一会，再按顺序下落。
- 拖尾图片素材可在编辑器“隐藏页彩蛋”里修改。
- 隐藏页保留返回按钮。

====================
十、常见问题
====================

Q1：为什么编辑器里填写了电脑本地路径，上传后图片不显示？

因为网页上线后不能读取你的电脑硬盘路径。请把图片放入 `website/assets/images/`，然后填写相对路径，例如：

`assets/images/avatar.png`

Q2：为什么修改后本地打开没变化？

可能是浏览器缓存。请尝试：

- 刷新页面；
- 使用 `Ctrl + F5` 强制刷新；
- 确认编辑器已经保存；
- 确认修改的是当前压缩包里的 `website/assets/js/site-data.js`。

Q3：为什么上传域名后仍然是旧版本？

请检查是否把 `website/` 文件夹里的内容上传到了域名根目录，而不是把整个 `website/` 文件夹套在根目录下面。

Q4：编辑器需要上传到域名吗？

不需要。`editor_tool/` 是本地工具，不建议上传到公开服务器。

Q5：隐藏页拖尾图片在哪里改？

打开编辑器，进入“隐藏页彩蛋”分类，修改拖尾图片路径即可。

Q6：音效在哪里改？

打开编辑器，进入“音效设置”分类。每个音效项后面都有预览按钮，可以先试听再填写路径。


====================
十一、上传前检查清单
====================

1. 网站首页能打开：`website/index.html`
2. 主页能打开：`website/home.html`
3. 图片路径不是电脑绝对路径。
4. 音频路径不是电脑绝对路径。
5. Godot iframe 路径指向 `assets/godot/你的游戏名/index.html`。
6. 简历图片和 PDF 路径已填写。
7. 隐藏页拖尾图片路径已填写或保持默认占位。
8. 编辑器里没有红点缺失提示，或确认缺失项是故意留空。
9. 上传服务器时只上传 `website/` 内部内容。
